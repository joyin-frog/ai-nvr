## 每次循环流程
期间不要询问我，不要切换其他plan mode 之类的，请自主优化整个项目
思考如何实现高帧率高清晰的原始的视频显示和处理，而非降低分辨率来解决卡顿问题，（ai相关处理或许可以有预先的降分辨率处理来减低cpu负担，但这个应该是事件可配置的）
### Step 1: 状态恢复
- 读 `EVOLVE_PROMPT.md`，有未完成任务 → 逐项执行
- 读 `PROJECT_STATE.md`，了解当前架构
- 都没有 → 进入 Step 3 自主完善

### Step 2: 执行任务
每项任务完成后必须：
1. 类型检查通过（`bunx tsc --noEmit` 或单文件检查）
2. 实机验证（后端用 `bun run dev` + curl；前端用 `cd web && pnpm dev` + pilot）
3. commit
4. 更新 `EVOLVE_PROMPT.md` 勾掉已完成项
然后继续下一项。全部完成后进入 Step 3。
